WST Final Project: 2-Week Sprint

Game Dev Web Portal & Original Mini-Game

Project Overview

This is a fast-tracked, 14-day development cycle for 2nd Year IT Game Development majors. Pairs will develop an original mini-game and host a professional GitHub Pages website to showcase their work. This project emphasizes clean UI, functional game loops, and collaborative version control.

The Goal: A functional 1-level game + A live 2-page website + Professional GitHub Repo.

Project Requirements

1. The Website (GitHub Pages)

  • Game Landing Page: A professional front-end design featuring game titles, instructions, and "Play Now" or "Download" links.
  • Development Blog/Team Page: A section documenting the team’s roles, the assigned genre, and links to both student GitHub profiles.
  • Responsive Design: The website must be navigable on both desktop and mobile browsers.

2. The Game (Any Platform)

  • Complete Game Loop: Must include a Start/Menu Screen, the core Playable Mechanic, and a Game Over/Score Screen.
  • Scope: Complexity should be simple and attainable within the two-week timeframe. Use of assets (sprites/audio) is allowed if properly credited.

3. GitHub Collaboration

  • Commit History: A shared repository with a minimum of 5 commits per student showing active collaboration.
  • Documentation: A professional README.md featuring a project description, controls, and high-quality screenshots.

Revised Project Schedule (2-Week Sprint)

Day 1-2: Setup & Planning

Genre assignment. Repository creation. Initial wireframing of the website and game logic.

Day 3-7: Core Development

Develop core game mechanics (movement, scoring, win/loss conditions) and build basic HTML/CSS site structure.

Day 8-11: Integration

Embed the game into the website portal. Finalize UI/UX styling and ensure GitHub Pages deployment is live.

Day 12-13: Testing & Debugging

Final bug fixes, peer testing, and finalizing the README file.

Day 14: Final Defense

Live 5-minute demonstration of the website and gameplay walkthrough.

Genre Assignment Pool

The professor will assign one unique genre per group. Descriptions are provided to help guide your design:

1. Platformer

Players navigate a character across suspended platforms and over obstacles using jumping mechanics. Success is measured by reaching the end of the level without falling or hitting hazards.

2. Tower Defense

Players must protect a base by strategically placing automated weapons along a path to stop waves of enemies. The game loop focuses on resource management and optimal tower placement.

3. Endless Runner

The character moves forward automatically, and the player must dodge obstacles or collect items to survive as long as possible. The primary goal is to achieve the highest score before the inevitable crash.

4. Puzzle Logic

Players solve non-action challenges that require pattern recognition or logical deduction to progress. This genre emphasizes "Aha!" moments through clever level design rather than reflexes.

5. Top-Down Shooter

Viewed from an aerial perspective, players control a character that moves and shoots in multiple directions to clear enemies. It requires responsive controls and basic enemy AI behaviors.

6. Clicker/Idle

Gameplay revolves around clicking an object to gain currency, which is then spent on upgrades to automate the process. It is a math-heavy genre that focuses on progression curves and exponential growth.

7. Turn-Based Combat

Players and enemies take turns selecting actions from a menu, such as attacking, defending, or using items. This allows for strategic decision-making without the pressure of real-time movement.

8. Rhythm Tap

Players must perform inputs in synchronization with a musical beat or visual cues. Accuracy and timing are the core metrics for determining the player's performance level.

9. Stealth Escape

Players must navigate through a level while avoiding the detection cones of guards or security cameras. The game ends if the player is spotted, forcing a focus on patience and timing.

10. Match-3

The objective is to swap adjacent tiles on a grid to create rows or columns of three identical items. Clearing tiles earns points and often triggers cascading effects for higher scores.

11. Racing/Time Trial

Players control a vehicle or character trying to complete a course in the fastest time possible. Success depends on mastering cornering and maintaining top speed throughout the track.

12. Visual Novel

This is a narrative-driven genre where players progress through a story and make choices that lead to different endings. It relies heavily on text, character art, and simple branching logic.

13. Bullet Hell

A subgenre of shooters where the screen is filled with intricate patterns of enemy projectiles. Players must have high precision to navigate through tiny gaps in the "curtain" of fire.

14. Tycoon/Manager

Players act as an administrator of a business or system, making financial decisions to grow their empire. The game loop involves balancing income against operational costs and upgrades.

15. Side-Scroll Brawler

Players move from left to right, fighting off groups of enemies using melee combat. It emphasizes simple combo systems and "locking" the screen until all current enemies are defeated.

16. Card Battler

Combat is resolved through the use of a deck of cards representing various abilities or creatures. Players must manage their hand and resources to outsmart the opponent's deck.

17. Memory Match

Players flip over pairs of hidden tiles to find matching sets within a limited number of moves or time. It is a straightforward test of short-term memory and concentration.

18. Physics Sandbox

The game provides a set of objects and tools that interact based on realistic physics like gravity and momentum. Players are given a specific goal to reach using creative construction or destruction.
